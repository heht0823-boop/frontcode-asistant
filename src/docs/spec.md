# FrontCode Assistant Spec

## 需求文档

- 这是项目的长期需求文档。
- 使用 `/spec 需求描述` 时，必须先根据当前需求改造本文件，再规划执行。
- 用户输入 `@` 或包含 `@文件名` 时，只弹出项目文件列表并选择文件，不直接请求大模型。
- 已选择的文件会暂存为下一次问题或 `/spec` 需求的上下文；再次输入 `@` 可以继续选择更多文件。

## 技术文档

- 项目使用 Node.js + ESM + JavaScript。
- 终端交互入口是 `src/app.js`。
- 命令和文件选择交互位于 `src/utils/interactive.js`。
- 主输入、命令选择和文件选择统一使用 inquirer 体系，避免 readline 与选择列表抢占 stdin 导致程序退出。
- 大模型请求位于 `src/request/askAI.js`。
- 用户历史保存位于 `src/utils/fshandle.js`。

## 规划规范

1. 先理解用户需求和已有文档。
2. 先更新需求文档和技术文档，再输出规划。
3. 如果规划存在错误、缺口或不确定点，返回问题并二次改造文档，不直接执行。
4. 只有规划通过校验后，才根据最终规划执行并输出结果。
